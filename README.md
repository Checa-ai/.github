## 📌 Sobre o Checaaí

O **Checaaí** é uma plataforma SaaS desenvolvida para transformar a maneira como empresas lidam com rotinas, padronizações e processos. 

Nossa missão é facilitar tanto a **criação estruturada** quanto a **execução prática** de checklists organizacionais. Tudo isso de forma fluida e acessível via aplicativo, garantindo que as equipes de campo e de gestão estejam sempre na mesma página.

> Menos tempo gerenciando papéis e planilhas perdidas, mais tempo focando em produtividade e qualidade.

---

## 🛠️ Nossa Stack de Tecnologia

Para entregar uma experiência de alto desempenho, que seja escalável e segura, construímos o Checaaí utilizando ferramentas modernas e robustas:

| Camada | Tecnologias Utilizadas |
| :--- | :--- |
| **Frontend** | Vue.js, JavaScript |
| **Backend** | Node.js, Express |
| **Banco de Dados** | PostgreSQL |
| **Armazenamento** | AWS S3 (Buckets para arquivos e mídias) |
| **Integrações** | Google APIs (Envio de e-mails, mapas, etc.) |

---

## 📂 Arquitetura e Repositórios

Nossa base de código é dividida estrategicamente para garantir a melhor manutenibilidade, separando as responsabilidades de interface, regras de negócio e infraestrutura. Aqui na organização, você encontrará:

*   **`checaai-frontend`**: A interface do usuário (Web e App), desenvolvida em Vue.js para garantir uma navegação reativa, leve e amigável.
*   **`checaai-backend`**: O coração do nosso sistema. Uma API REST construída com Node.js e Express, responsável por toda a lógica de checklists, usuários e permissões.
*   **`checaai-webapp`**: A interface do app (Web), desenvolvida em Vue.js para garantir uma navegação reativa, leve e amigável.

## ⚙️ Primeiros Passos (Para Desenvolvedores)

Se você acabou de chegar ao time, seja muito bem-vindo(a)! Para começar a codar e rodar o ambiente localmente, siga este checklist rápido:

1. **Clone os repositórios** nos quais você irá trabalhar.
2. **Solicite as variáveis de ambiente (`.env`)** ao administrador do sistema. Você precisará das credenciais do PostgreSQL, chaves de acesso da AWS e tokens do Google.
3. **Instale as dependências** rodando `npm install` ou `yarn install` na raiz de cada projeto.
4. **Configure o banco de dados** garantindo que o serviço do Postgres esteja rodando localmente na sua máquina.
5. **Suba as aplicações** utilizando o comando `npm run dev` ou `yarn dev` (consulte o README específico de cada repositório para comandos detalhados).
